# Manual Test Plan

このリポジトリで行った変更の簡易テスト手順。

前提: 仮想環境を使い、`app.py` を起動して操作する。

1. 初回起動テスト
   - すべてのプロセスを終了し、`app.py` を起動する。
   - DB が空の場合は自動でサーバー管理ダイアログが開くこと。
   - サーバーを1つ作成し、保存する。
   - メイン画面でサーバーの名前・バージョン・ソフト・プラグインフォルダが反映されること。

2. 再起動後の永続性
   - アプリを終了して再起動する。
   - メイン画面のサーバー選択肢に作成したサーバーが表示されること。
   - `scripts/check_db.py` を実行して DB とサーバーDB状態を確認する。

3. 新規作成と上書きの検証
   - 既存サーバーが選択された状態で「サーバー管理」を開き、`新規作成` を押す。
   - フォームに入力し `保存して追加` で新規サーバーが追加されること。
   - 既存サーバーが上書きされないこと（名前が変わらず既存の値が残るべきでない）。

4. プラグイン登録時の保護
   - サーバー未選択の状態で `取り込み` / `スキャン` を試すと警告が出て処理が中断されること。

5. マイグレーション検証
   - `master` の `plugins` 行が存在する場合、`open_server_db()` 実行後に該当行が server DB に移動し、master 側から削除されること（手動で検証）。


コマンド例:

```powershell
# 仮想環境がアクティブなら
py -3 -m venv .venv
.\.venv\Scripts\activate
py -3 app.py
```

```powershell
# DB チェック
py -3 scripts\check_db.py
```
